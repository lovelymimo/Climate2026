# Climate Safety Hub - 개발 플랜

## 1. 개발 순서
- **백엔드는 맨 마지막에 개발**
- 프론트엔드 완성 후 백엔드 연동 진행

## 2. 프론트엔드 개발 원칙
- **프론트엔드만 우선 구축**
- 데이터 저장: `localStorage` 기반으로 동작
- 실제 데이터가 필요한 경우: **더미데이터(Mock Data)** 활용
- API 연동 부분은 인터페이스만 준비해두고, 실제 연동은 백엔드 완성 후 진행

## 3. MVP 핵심 화면 (5개)
모든 화면을 만들지 않고, **핵심 5개 화면**만 우선 개발:

| 순서 | 화면명 | 설명 |
|------|--------|------|
| 1 | 홈/스플래시 | 서비스 소개, 메인 진입점 |
| 2 | 침수 위험 지도 | 경기도 지도 + 위험 영역 시각화 + 지역 선택 |
| 3 | 지역 침수 안내 | 선택한 지역의 위험 정보 + 행동 가이드 |
| 4 | 시민 제보 | 위치 선택 + 제보 유형 + 설명 입력 |
| 5 | 제보 완료/확인 | 제보 성공 안내 + 내 제보 확인 |

## 기술 스택
- **프레임워크**: React + Vite
- **지도 API**: Kakao Maps API
- **상태 관리**: React Context + localStorage
- **스타일링**: Tailwind CSS
- **언어**: TypeScript
