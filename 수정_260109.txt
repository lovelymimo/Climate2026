너는 React + Vite + TypeScript 프론트엔드 엔지니어다.
내 프로젝트는 로컬에 이미 존재하며 VSCode에서 UI/레이아웃을 여러 번 수정했다.

중요:
- 반드시 현재 워킹트리의 최신 파일을 직접 열어 “현재 내용”을 확인한 뒤 패치를 적용해라.
- 과거 대화나 추측으로 코드를 작성하지 마라.
- 기존 디자인(특히 src/index.css의 cs-* 클래스 및 헤더/메뉴 UI)을 유지하고, 필요한 최소 변경만 하라.

[문제 요약]
현재 ‘침수 위험 지도(MapPage)’에서 다음 문제가 있다.
1) “지역을 선택하면 … 지도에서 확인합니다”라고 안내하지만, 실제로
   - 지역검색 후 “찾기(검색 실행)” 버튼이 없거나
   - 결과 리스트/지도 반영이 동작하지 않는다.
2) 우측 “시/군 선택”, “구/읍면동 선택” 드롭다운이 비어있거나,
   경기도 전체 시군 리스트가 정상적으로 표시되지 않는다.
3) 지도 하단의 “침수위험도 / 배수취약도 / 과거 피해” 카드가
   실제 데이터 근거가 불명확하며, 현재 상태가 ‘연결된 실데이터’인지 확실치 않다.
   → 정확한 데이터에 근거한 값만 표시하도록 연결하거나,
     연결이 없다면 ‘미연결/근거없음’ 상태로 명확히 표시하고 placeholder 값은 제거해야 한다.

[프로젝트 목표]
- MapPage에서 “지역 검색/선택 → 지도 이동(zoom/pan) → WMS 오버레이 확인” 흐름을 완성한다.
- 시/군/구(읍면동) 선택 UI가 실제로 동작하고, 최소한 경기도 시/군 목록이 정상 표출되게 한다.
- 하단 통계 카드 3개는 ‘근거 있는 데이터 소스’에 연결되었을 때만 숫자를 표시한다.
  연결 전에는 사용자에게 명확히 “데이터 미연결” 안내를 보여라(거짓 숫자 금지).

[구현 요구사항]
A) 지역 검색/선택 UX 완성
- 검색 입력(지역 검색) + “찾기” 버튼을 제공한다(엔터키도 동작).
- 검색 성공 시:
  - 선택된 지역 텍스트가 “선택 지역: …”에 반영
  - 지도는 해당 지역 중심으로 이동/적절한 zoom 적용
  - WMS 오버레이는 기존 탭/슬라이더 설정을 유지한 채 표시
- 검색 실패 시: 결과 없음 안내(토스트/인라인) + 지도는 그대로 유지

B) 행정구역 드롭다운 정상화
- 최소 요구: 경기도 시/군 전체 리스트가 “시/군 선택”에서 보이게 할 것.
- “구/읍면동 선택”은
  - 데이터 소스를 확보할 수 있으면 의존형(시/군 선택 후 하위 목록 표시)으로 구현
  - 소스가 없으면 UI는 유지하되 “데이터 준비중”으로 명확히 표시(빈 드롭다운 금지)

※ 데이터 소스 원칙(중요)
- 시/군/읍면동 목록은 신뢰 가능한 소스에 근거해야 한다.
- 내가 별도 API 키를 주지 않았으므로,
  1) 우선은 ‘경기도 시/군 목록’을 프로젝트 내부 정적 데이터(JSON)로 넣어 동작시키고,
  2) 읍면동까지는 “데이터 소스가 있는 경우에만” 연결하되, 없으면 ‘준비중’으로 표기.
- 정적 데이터는 /src/data/ 경로에 분리하고, 컴포넌트에 하드코딩하지 마라.

C) 하단 통계 카드 3개 데이터 정리(거짓 데이터 제거)
- 침수위험도/배수취약도/과거피해 숫자/등급이 현재 임의값이면 즉시 제거하고,
  “데이터 미연결” 또는 “근거 데이터 연결 필요” 상태 UI로 교체한다.
- 실데이터 연결이 가능한 경우에만 표시한다.
  (예: WMS/별도 API/정적 통계 데이터 등 실제 근거)
- 최소 변경 원칙을 지키되, 사용자 신뢰를 해치는 가짜 수치는 금지.

D) 기술/품질 요구
- 현재 디자인(cs-* 클래스)과 레이아웃을 유지하면서 기능만 자연스럽게 붙인다.
- 반응형(모바일/태블릿)에서 UI가 깨지지 않게 한다.
- 로딩/실패 상태를 명확히:
  - 검색 중 로딩
  - WMS 레이어 로딩 실패(네트워크/CORS 포함) 안내
- 콘솔 에러/타입 에러 없이 빌드 가능해야 한다.

E) CORS 정책
- WMS 타일이 CORS로 막히면 vite.config.ts에 개발 프록시 옵션을 추가하라.
  단, “실제로 CORS 에러가 확인될 때만” 적용하도록 설계하고,
  기본값은 직접 연결 시도(프록시 불필요 시 건드리지 않기).
  예: /ggwms -> https://climate.gg.go.kr/ols/api/geoserver
  프론트에서는 VITE_GG_WMS_BASE_URL=/ggwms/wms 형태로 전환 가능하게 정리하라.

[Acceptance Criteria]
- 시/군 선택 드롭다운에 경기도 시/군 목록이 정상 표시된다.
- 지역 검색 입력 + 찾기 버튼이 존재하고, 검색 실행 시 결과/선택 지역/지도 이동이 실제로 동작한다.
- 하단 3개 카드에 근거 없는 숫자는 표시되지 않는다(미연결 상태 UI로 처리).
- 지도 탭(WMS 4개 전환)과 투명도 슬라이더는 기존처럼 정상 동작한다.
- 빌드/타입 에러 없음.
